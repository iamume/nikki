# 03

昨日の夜は、ニッキを書くのに便利なワークフローを整備し始めました。いままでは Github Codespaces で書いていたのですが、使い慣れた iA Writer で書きたいなと思ってチョロっと検索したらそれっぽいやり方が出てきたので試してみました。

具体的には、iPad 用の Git クライアントである Working Copy と iA Writer を連携させるやり方です。フォルダ単位で sync を設定してあげる感じで、iA Writer でなんか書いたら Working Copy の方に勝手に同期されて、それをレポジトリに反映させるみたいな運用が出来るようになりました。便利ですね。

## 今後の予定

ただテキストを書き溜めるだけでも良いのだけど、適当に静的サイトジェネレーターでも作って、Github Pages だっけ、アレでギャンして 0xcc.jp にギャン出来たらベストだなーみたいなことは考えてます。Github Pages じゃなくても、0xcc.jp のサーバーで一日一回クロンタブでも回して同期してやれば良いんだけど、なんかこう、もうサーバーの管理が面倒くさいんだよね。自由度があるわけでもないサーバーにお金を払い続けるのもなんか馬鹿らしいし、VPS 借りて堅牢なサーバー構築するほどの情熱もないし、Github Pages が最適解って気がする。

インターネットとかプログラミングとかのトレンドを全然追いかけてないから、今一番イカすやりかたがどんなのか全然わからんのよね。一日二日かけて調べれば追いつくと思うんだけど、そこから必要な知識を雑に把握してドゥーする情熱はもうない。オートメーションの仕組みとかありそうなんだけどね。うーん。

別に全世界に公開してガン見されたいみたいな欲望があるわけでもないから、こんな感じでも別に良いんだけど。
